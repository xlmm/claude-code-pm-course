# Free Codex Tutorial for Product Managers

![Codex for Product Managers](/images/codex-banner.png)

## Learn Codex IN Codex!

This course is taught entirely inside Codex itself 😈

- You'll install Codex, download the course files, then start the first lesson — and **Codex becomes your instructor**
- It guides you through each module interactively, right in your terminal
- You learn by doing real PM work with real files

## Who This Course Is For

**Product Managers** who want the most powerful way to put AI to work — not as a chat window you copy-paste into, but as an agent that actually does the work alongside you.

OpenAI Codex isn't just for engineers. For PMs it's an AI workspace that:
- Reads your research, drafts and edits PRDs, and analyzes data across many files
- Runs multi-step tasks and even works in parallel on batches of work
- Remembers context about your product through a project file (`AGENTS.md`)
- Acts as custom reviewers (engineer, exec, UX) before you share work with your team

**You don't need coding or terminal experience.** If you can describe what you want in plain English, you can use Codex.

## What You'll Learn

### [Module 1: Fundamentals](/fundamentals/welcome) (~3 hours)

- **File operations:** Read, transform, and create documents with file references
- **Visual workspace:** Set up a split-screen workflow to see files as Codex works
- **Parallel agents:** Launch multiple Codex instances to process files simultaneously
- **Custom sub-agents:** Create permanent AI advisors with different perspectives
- **Project memory:** Use `AGENTS.md` so Codex remembers your product context
- **Commands & shortcuts:** Slash commands, keyboard shortcuts, power user features

### [Module 2: Advanced PM Work](/advanced/write-prd) (~2 hours)

- **Write a PRD:** Partner with Codex to write better PRDs faster with multi-perspective review
- **Analyze data:** Funnel analysis, impact estimation, and A/B test readouts
- **Product strategy:** Make hard strategic calls and build executive-ready slides

### [Module 3: Vibe Coding](/vibe-coding/setup) (~1-2 hours)

- **Setup & mindset:** The vibe coding approach — you describe, Codex builds
- **Planning:** Interview process to define clear requirements before building
- **Build & iterate:** Scaffold your app and refine it with screenshots
- **Version control:** Back up your work with GitHub
- **Go live:** Deploy to Vercel with a real URL anyone can visit
