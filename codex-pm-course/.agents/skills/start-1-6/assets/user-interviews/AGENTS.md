# User Interview Guidance

## Purpose

This file gives Codex folder-specific guidance for work inside `user-interviews/`.

## How to Handle Interview Notes

- Preserve direct quotes when they are useful.
- Separate what users said from what we infer.
- Group findings by theme, frequency, and severity.
- Cite the interview file when summarizing evidence.
- Avoid overgeneralizing from one interview.

## Output Preference

When synthesizing interviews, use this structure:

1. Key themes
2. Supporting quotes
3. Product implications
4. Open research questions
