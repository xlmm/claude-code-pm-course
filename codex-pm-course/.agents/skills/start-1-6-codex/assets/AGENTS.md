# TaskFlow Project Map

## Purpose

This file is a lean project map for Codex. Keep it short because Codex loads it as project context when working in this repo.

Use this file to point Codex toward the right sources of truth, stable terminology, and working conventions. Do not paste the whole company wiki here.

## Project Context

- Product: TaskFlow, a project management SaaS for remote-first teams
- Role: Senior PM for activation and onboarding
- Current product focus: improve new-user activation and time to first useful workflow

## Sources of Truth

- Company overview: `company-context/COMPANY.md`
- Product overview and terminology: `company-context/PRODUCT.md`
- User personas: `company-context/PERSONAS.md`
- Writing conventions: `company-context/WRITING-STYLE.md`
- User interview files: `user-interviews/`

## Stable Conventions

- Use `TaskFlow` capitalization.
- Use `Workspace`, `Task`, and `Epic` as product terms.
- Prefer concise bullets for summaries.
- For product docs, include user impact, success metrics, and open questions.
- When making claims from research, cite the relevant source file.

## Folder-Specific Guidance

Check for additional `AGENTS.md` files in subfolders before working deeply in that area. For example, `user-interviews/AGENTS.md` explains how to handle research notes.
